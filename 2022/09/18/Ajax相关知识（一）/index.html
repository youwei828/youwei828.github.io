Ajax基本知识（一）

2022/9/18 Ajax基本知识 JavaScript

一、Ajax是什么

Ajax是Asynchronous JavaScript and XML（异步JavaScript和XML）的简写

Ajax中的异步：可以异步的像服务器发送请求，在等待请求响应的过程中，不会阻塞当前的页面，浏览器可以做自己的事情。直到成功获取数据后，浏览器才开始搞处理响应的数据。

XMl（可扩展标记语言）是前后端数据通信时传输的一种数据格式。目前XML已经不是很常用，现在比较常用的是JSON。

Ajax其实就是浏览器与服务器之间的一种异步通信方式。

使用Ajax可以在不重新加载整个页面的情况下，对页面的某部分进行更新。

二、搭建Ajax开发环境

Ajax需要服务器环境，非服务器环境下，很多浏览器无法正常使用Ajax。

NOTE：本地安装服务器的方法有很多种，这里选择最简单的一种，使用VS code编辑器下载Live server插件，安装好插件后重启一下，之后在VS code编辑html代码页面右击点击Open with live server，或者快捷键ALT+L ALt+O。

还可以使用windows下的PHPstudy或者Mac下的MAMP。

三、XMLHttpRequest

Ajax想要实现浏览器与服务器之间的异步通信，需要依赖XMLHttpRequest，它是一个构造函数 。

不论是XMlHttpRequest，还是Ajax，都没有具体的某种数据格式绑定。

四、Ajax的使用步骤

1、创建xhr对象

const xhr = new XMLHttpRequest();

2、监听事件，处理响应

当获取到响应后，会触发xhr对象的readystatechange事件，可与在该事件中对响应进行处理，readystatechange事件监听readyState这个状态的变化，它的值从0-4，一共五个状态，每一次变化状态都会触发这个事件。

0：未初始化。尚未调用open()；

1：启动。已经调用open(),但尚未调用send();

2：发送。已经调用send()，但未收响应；

3：接收。已经接收到部分响应数据；

4：完成。已经接收到全部的响应数据，而且已经可以在浏览器中使用了。

xhr.onreadystatechange=()=>{
    if(xhr.readState!==4)
        return;//把不等于4的状态返回，之后查看状态码HTTP CODE；
    //获取到响应后，响应的内容会自动填充到xhr对象的属性中；
    if((xhr.status>=200)&(xhr.status<300)||(xhr.status===304){
       console.log('正常使用响应数据')}
      )else{
          '有错误分情况进行处理'}
}
    //xhr.status表示HTTP的状态码(200、404)。
    //xhr.statusText表示HTTP状态码说明(OK、Not Found)。
    //xhr.responseText表示以字符串类型存储的响应数据。

3、准备发送请求

open方法有三个参数：

xhr.open[‘HTTP方法GET、POST、PUT、DELET’，’地址URL，可以远程地址 也可以是本地地址 例如./index.html ./index.xml index.txt’,’true为异步 false为同步（很少用同步）’]

调用open并不会真正大发送请求，只是做好发送请求前的准备工作。

4、发送请求

调用send()正式发送请求，send()的参数是通过请求体携带的数据。

xhr.send(‘’) ，如果open请求使用的GET，就不需要传数据，填入null；相反使用POST请求就需要传数据。

=======================================================================================

NOTE：readystatechange事件也可以支持addEventListener使用，不过要注意，IE6~8不支持addEventLisent.

为了兼容性，readystatechange中不使用this，而直接用xhr变量名。

xhr.addEventListener('readystatechange',()=>{},false)

状态码如下：

分类分类描述
1**信息，服务器收到请求，需要请求者继续执行操作
2**成功，操作被成功接收并处理
3**重定向，需要进一步的操作以完成请求
4**客户端错误，请求包含语法错误或无法完成请求
5**服务器错误，服务器在处理请求的过程中发生了错误
